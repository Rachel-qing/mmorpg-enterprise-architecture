# 🎮 MMORPG Enterprise Architecture

## 📌 Overview

Designed an end-to-end enterprise architecture for an MMORPG company, connecting business strategy, operational processes, data models, and IT systems to enable scalable and data-driven decision-making.

---

## 🧩 Architecture Design

### Business Layer

* Goal model linking strategic objectives to operational targets
* VDML-based value exchange across players, suppliers, and partners

### Process Layer

* EPC models for core workflows:

  * Subscription management
  * In-game transactions
  * Supplier onboarding

### Data Layer

* REA conceptual schema capturing key economic events
* Structured relationships across customers, company, and partners

### System Layer

Integrated enterprise systems:

* ERP (payments, subscriptions)
* CRM (player lifecycle)
* BPMS (process orchestration)
* Game Server (activity data)

→ Unified via ETL into a centralized Data Warehouse

---

## 📊 KPI Framework

Designed KPI system aligned with business goals:

* **Leading:** MAU, churn rate, CSAT, feature releases
* **Lagging:** revenue, market share, cost per player

All KPIs are mapped to processes and supported by defined data sources.
